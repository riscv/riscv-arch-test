==== Misalign: Misaligned Loads/Stores

Misaligned loads/stores to main memory optionally can be handled in hardware.  When the UDB MISALIGNED_LDST parameter is true, misaligned scalar loads and stores to main memory do not throw an exception.  This parameter is a simplification of the "Access Type PMAs" which could define misaligned behavior on each memory region.

The misaligned family of tests exercises integer, compressed, and single/double/quad-precision floating-point loads/stores.  It uses the new cp_misalign coverpoint defined in <<t-Misaligned-modified-coverpoints>>. It assumes that if these work, then Zfh and Zcb loads/stores will also work correctly. ==== Zicclsm Misaligned Loads/Stores

The Zicclsm extension, defined in the https://drive.google.com/file/d/1Kg7Ner5ZlxFDclf92-9Tz88JvmZWt5Wb/view[RVA22] and
https://github.com/riscv/riscv-profiles/blob/main/src/rva23-profile.adoc[RVA23] profiles, provides the following normative rule.  The extension applies to privilege modes below M-mode (because the profiles apply to supervisor-mode execution environment), and requires that misaligned scalar and vector loads and stores to main memory either are handled in hardware or raise a misaligned exception that is handled by the execution environment.  This is tested by Exceptions coverpoints that either return the expected value or throw a misaligned trap. Misaligned atomics are not part of Zicclsm, but are checked in {ExceptionsZaamo, ExceptionsZalrsc}/{cp_load_address_misaligned, cp_store_address_misaligned}.

|zicclsm_op |Misaligned loads and stores to main memory regions with both the cacheability and coherence PMAs must be supported. |{ExceptionsS, ExceptionsU, ExceptionsZc, ExceptionsF, ExceptionsVls}/{cp_load_address_misaligned, cp_store_address_misaligned}
|===

==== Zicntr Counters

Zicntr provides unprivileged read access to the `cycle`, `time`, and `instret` counters.
Note that, like all other unprivileged tests, these tests are performed in M-mode.
The <<t-ZicntrS-coverpoints, ZicntrS>> and <<t-ZicntrU-coverpoints, ZicntrU>> privileged tests exercise reading the counters in S-mode and U-mode,
with control by `mcounteren` and `scounteren` CSRs.
The <<t-Sm-coverpoints, Sm>> privileged tests exercise `mcycle`, `minstret`, and HPMs from M-mode.

[[t-Zicntr-coverpoint-definition]]
.Zicntr Coverpoint Definition
[cols="2, 1, 4" options=header]
|===
|Coverpoint|Bins|Definition
|cp_cntr|3|Read each of the counters (`cycle`, `time`, `instret`) twice using `csrrs rd, csr, x0` and check that the difference is exactly right for `instret`, or at least 1 for `cycle`, 0 for `time` (`time` may not increment between reads).
For RV32, also check `cycleh`, `timeh`, `instreth` with the same definitions.
Skip `time` and `timeh` if parameter TIME_CSR_IMPLEMENTED = False.
|===

*UDB Parameters:* TIME_CSR_IMPLEMENTED defines whether the `time` and `timeh` CSRs are implemented.  If False, the cp_cntr coverpoint skips checking `time` and `timeh`.

==== Zihpm Hardware Performance Monitors

Zihpm provides unprivileged read access to custom hardware performance monitor CSRs.

[[t-Zihpm-coverpoints]]
.Zihpm Instruction Coverpoints
include::{testplansdir}/Zihpm.adoc[]

include::norm/Zihpm_norm_rules.adoc[]

[[t-Zihpm-modified-coverpoints]]
.Zihpm Coverpoint Definition
[cols="2, 1, 4" options=header]
|===
|Coverpoint|Bins|Definition
|cp_cntr_hpm|29|Read each of the HPMs twice and check that the difference is non-negative.  For RV32, also check the high halves of each HPM.
|===

Note that this coverpoint assumes that reading hpmcounter* never traps.
See https://github.com/riscv/riscv-isa-manual/issues/2558[RISC-V ISA Manual Issue #2558] for discussion.
There may not be a graceful way to certify Zihpm on an implementation that allows hpmcounter* reads to
sometimes trap because unprivileged tests should not trap.

*UDB Parameters:* None.  MCOUNTENABLE_EN, SCOUNTENABLE_EN, HCOUNTENABLE_EN, exercised by ZicsrS/U/H cp_xcounteren_access_y tests (<<t-ZicntrS-coverpoints>>). COUNTINHIBIT_EN is exercised by Sm/cp_inhibit tests (<<t-Sm-coverpoints>>).
